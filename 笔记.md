git 分布式版本控制系统
从个人开发到团队协作

### 三大区域
    工作区、暂存区、本地库、远程库
    工作区 git add =>  暂存区
    暂存区 git commit =>  本地库

#### 工作区
    写代码，还未保存
#### 暂存区
    代码已经临时保存下来，但还没有提交上传到库中
#### 本地库
    代码最终保存的地方，作为历史版本（不能删除！！！）
#### 远程库
    代码托管中心，如github，gitee等，局域网：gitlab

#### 常用命令
    git config--global user.name 用户名          设置用户签名
    git config--global user.email 邮箱           设置用户签名
    git init                                    初始化本地库        （生成.git文件夹）
    git status                                  查看本地库状态
    git add 文件名                              添加至暂存区
    git commit -m '日志文件' 文件名             提交到本地库
    git reflog                                  查看历史记录（本地库）
    git reset --hard 版本号                           版本穿梭

    git rm --cached 文件名                      从暂存区中删除文件 
    git log                                     查看详细的历史记录

#### 分支
    类似于副本（底层本质还是指针的指向）
    可以进行多方面并行开发，互不影响
##### 分支相关命令
    git branch  分支名          创建分支
    git branch -v               查看分支
    git checkout 分支名         切换分支
    git merage  分支名          把指定的分支合并到当前分支上

##### 分支冲突
    合并分支时如果存在两个不同分支同时对某个文件进行修改等情况会出现分支冲突
    此时需要人为去处理，并重新add commit 文件 注意：commit时不需要写上文件名

#### 远程库
    本地库 （push）推送  远程库     （将自己的本地库数据传给远程库）
    远程库 （clone）克隆  本地库    （本地库没有数据，获取全部的远程库数据）
    本地库 （pull）拉取   远程库    （本地库有数据，将远程库的数据和自己的合并起来，即获取最新的数据到自己本地库上）

    跨团队协作时，团队外的人要先fork（类似于分支）得到团队的远程库到自己的远程库中，再在自己的本地库中进行修改等，完成后
    再推送到自己的远程库，给团队的远程库发送推送请求，得到运行后，团队的远程库将和分支进行合并

#### 命令
    git remote -v       查看当前所有远程地址别名
    git remote add 别名 远程地址        起别名
    git push 别名 分支          推送本地分支上的内容到远程仓库
    git clone 远程地址          将远程库的内容克隆到本地
    git pull 远程库地址别名 远程分支名      将远程库对于最新内容拉下来和当前本地分支合并
